# LS-2 Runbook: LangSmith Tracing for Blush Hour

## A) Overview

Purpose:
- Enable safe observability for Blush Hour so we can inspect both API root requests and nested LLM chain activity.
- Confirm LS-1 API root traces are emitted and LS-3 smoke verification can be run repeatedly.

What you'll see:
- Chain runs: nested model/tool/span activity generated by LangChain/LangGraph instrumentation.
- API root runs: top-level FastAPI request traces using the LS-1 naming convention `api {METHOD} {PATH}`.

Common API root names to filter:
- `api POST /api/chat-night/icebreakers`
- `api POST /api/chat-night/icebreakers/reveal`

## B) Required env vars (placeholders only)

Set these in `.env` at repo root using placeholders only:

```dotenv
LANGSMITH_TRACING=true
LANGSMITH_API_KEY=ls__...
LANGSMITH_PROJECT=default
BH_LANGSMITH_API_TRACING_ENABLED=true
OPENAI_API_KEY=sk__...
CHAT_NIGHT_ICEBREAKERS_PROVIDER=openai
CHAT_NIGHT_ICEBREAKERS_MODEL=gpt-4o-mini

CHAT_NIGHT_ICEBREAKERS_MAX_CALLS_PER_DAY=100
CHAT_NIGHT_ICEBREAKERS_MAX_CALLS_PER_USER_PER_DAY=50
CHAT_NIGHT_ICEBREAKERS_MAX_CALLS_PER_ROOM=10
CHAT_NIGHT_ICEBREAKERS_MIN_SECONDS_BETWEEN_OPENAI_CALLS=0
```

Optional compatibility vars:

```text
LANGCHAIN_TRACING_V2=true (optional)
LANGCHAIN_API_KEY=<same as LANGSMITH_API_KEY> (optional)
```

Notes:
- `LANGCHAIN_TRACING_V2` and `LANGCHAIN_API_KEY` are optional compatibility toggles.
- Keep keys in local `.env` only. Do not place real values in docs, commits, or screenshots.

## C) .env location + ignore

- `.env` must live at repo root: `C:\001_dating_app\blush_hour_v0\.env`
- Ensure `.env` is git-ignored before running tracing:

```powershell
git check-ignore -v .env
```

- Expected: output shows `.gitignore` rule covering `.env`.

## D) PowerShell .env loader snippet

Load `.env` into process environment in the same terminal session where backend will start:

```powershell
Get-Content .env | ForEach-Object {
  $line = $_.Trim()
  if (-not $line -or $line.StartsWith("#")) { return }
  $pair = $line -split "=", 2
  if ($pair.Count -ne 2) { return }
  [Environment]::SetEnvironmentVariable($pair[0].Trim(), $pair[1].Trim().Trim('"'), "Process")
}
```

## E) Start backend (must be same terminal)

After loading env in terminal A, start backend in terminal A:

```powershell
cd backend
venv\Scripts\python.exe -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

Important:
- If `uvicorn` is started in a different terminal than the `.env` loader step, tracing env vars are not present and LangSmith traces will not appear.

## F) Verify tracing (LS-3)

Run contract + smoke verifiers from repo root:

```powershell
backend\verify_chat_night_icebreakers_contract.ps1
backend\verify_langsmith_api_tracing_smoke.ps1 -BaseUrl "http://localhost:8000"
```

Expected smoke verifier proof line:
- `PASS: langsmith api tracing smoke`

LangSmith UI steps:
1. Open LangSmith and select project from `LANGSMITH_PROJECT` (default: `default`).
2. Filter by:
   - `api POST /api/chat-night/icebreakers`
   - `api POST /api/chat-night/icebreakers/reveal`
3. Confirm newest runs align with verifier execution time and include nested chain activity.

## G) Troubleshooting

0 traces checklist:
- Confirm backend terminal has `BH_LANGSMITH_API_TRACING_ENABLED=true`.
- Confirm one tracing toggle is true: `LANGSMITH_TRACING=true` or `LANGCHAIN_TRACING_V2=true`.
- Confirm `LANGSMITH_API_KEY` is present in server terminal process.
- Confirm LangSmith UI project matches `LANGSMITH_PROJECT`.
- Confirm you are filtering with LS-1 root names (`api {METHOD} {PATH}`).

fallback model checklist:
- Guardrails too strict: verify temporary test overrides are set exactly as in Section B.
- Wrong terminal: verify `OPENAI_API_KEY` and provider vars are loaded in the same terminal that launched `uvicorn`.
- Missing key: confirm `OPENAI_API_KEY` exists and `CHAT_NIGHT_ICEBREAKERS_PROVIDER=openai`.

Safety rules:
- Never commit `.env`.
- Never paste real keys into docs, logs, tickets, or PR comments.
- Tracing allowlist only: trace chat-night endpoints only; do not trace auth/OTP endpoints.
- Confirm tracing inputs/metadata are PII-safe before sharing screenshots or links.
